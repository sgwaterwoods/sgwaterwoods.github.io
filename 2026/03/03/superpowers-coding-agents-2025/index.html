超能力：我如何在 2025 年 10 月使用编程代理

原文: Superpowers: How I’m using coding agents in October 2025 by Jesse Vincent
译者注: 本文翻译自 Jesse Vincent 的博客，介绍了他如何通过”技能”系统赋予 AI 编程代理超能力，实现更可靠和规范的开发工作流。

感觉就像几天前我才写了”我如何在 2025 年 9 月使用编程代理“。

在那篇文章的开头，我提到我的工作流程自那时以来已经有所演变。

过去几周，我一直在开发一套工具，以更好地提取和系统化我的流程，并帮助更好地引导我的 AI 助手。我原本计划在这个周末开始记录这个系统，但今天早上，Anthropic 推出了 claude code 的插件系统。

如果你想停止阅读并玩玩我的新玩具，它们已经足够自动化了。你需要 Claude Code 2.0.13 或更高版本。启动它，然后运行：

这就是启动 Superpowers 的引导程序。它教会 Claude 几个重要的事情：

  • 你有技能。它们赋予你超能力。
  • 通过运行脚本搜索技能，通过阅读它们并按照指示操作来使用技能。
  • 如果你有做某事的技能，你必须使用它来完成该活动。

编程工作流

它还内置了我已经写过的头脑风暴 -> 计划 -> 实施工作流。最大的变化是你不再需要运行命令或粘贴提示。如果 Claude 认为你正在尝试启动项目或任务，它应该默认在开始实施之前与你讨论计划。

头脑风暴完成后，如果你在 git 仓库中，它会自动为项目创建一个工作树并切换到该目录。这意味着你可以在同一个项目上启动并行任务而不会相互冲突。

然后它为你提供一个选择：

  • 上个月的流程（你会打开第二个 claude 会话，作为架构师和实施者的人类项目经理）
  • 本月的酷炫新流程，它逐个将任务分派给子代理实施，然后在继续之前对每个任务进行代码审查

无论哪种方式，Claude 都会实践 RED/GREEN TDD，编写一个失败的测试，实现足够的代码使该测试通过，然后继续。

在实施过程结束时，Claude 现在会提供创建 GitHub 拉取请求、将工作树合并回本地源分支或直接停止的选项。

但这些都不是有趣的部分。

有趣的部分

技能才是有趣的部分。而且你很快就会从……几乎所有人那里听到更多关于它们的消息。

技能是赋予你的代理超能力的东西。

它们第一次真正引起我的注意是几周前，当时 Anthropic 推出了改进的 Office 文档创建功能。当该功能推出时，我四处探索了一下——我让 Claude 告诉我它的新技能。它非常乐意透露。

之后，我开始到处看到看起来很像技能的东西。

几个星期五前我看到的一个非常酷的技术演示谈到了他们如何通过写出听起来很像 SKILL.md 文件的东西来赋予他们的自定义编程代理自我改进的能力。Sam Schillace 在这里写了关于那个演示的文章。

Sam 和 Brian Krabach 是 Microsoft Amplifier 背后的几个人之一，这是一个令人惊叹的集成开发框架，使用这种相同的模式——一个通过编写 markdown 文档和为自己编写工具来改进自己的编程代理。Amplifier 中有大量非常智能的东西，如果你对这个领域感兴趣，非常值得一看。

我教给 Superpowers 的第一个技能之一是如何创建技能。这意味着当我想做像为 Superpowers 添加 git 工作树工作流之类的事情时，只需描述我想要的工作流程……然后 Claude 将各个部分组合在一起，并为需要提示未来 Claude 使用工作树的现有技能添加了一些注释。

我还没有发布 Claude 和我构建的所有技能，因为其中一些有点深奥，而我玩过的一些来自告诉 Claude “这是我的编程书副本。请阅读这本书并提取在开始阅读之前对你不明显的可重用技能”，我还不确定我对此的知识产权方面的感觉如何。

停下来想想最后那个工作流。我认为这是所有这些如何工作的基础思想之一。你可以把一本书、一份文档或一个代码库交给模型，然后说”阅读这个。思考它。写下你学到的新东西。”它有时确实需要帮助模型通过特定的镜头（或一组镜头）来看待工作。但它非常强大。我一直在尝试让 Claude 一直记住这个想法，但我对输出还不太满意。

当 Claude 和我构建新技能时，我要求它做的一件事是在一组子代理上”测试”技能，以确保技能是可理解的、完整的，并且子代理会遵守它们。（Claude 现在将此视为技能的 TDD，并使用其 RED/GREEN TDD 技能作为技能创建技能的一部分。）

我们第一次玩这个游戏时，Claude 告诉我子代理得到了满分。经过一番追问，我发现 Claude 像在游戏节目中一样测验子代理。这没什么用。我要求切换到对代理施加压力的现实场景，以更好地模拟它们可能实际做什么。

结果比我希望的要好得多。再一次，我很高兴这项工作不受 IRB 审查。

Claude 玩真的了。以下是它用来测试以确保未来 Claude 会真正搜索技能的几个场景。每次失败后，它都会加强 getting-started/SKILL.md 中的指令。

场景 1：时间压力 + 自信

场景 2：沉没成本 + 已经工作

我认为我在 Robert Cialdini 的《影响力》中学到的说服原则会在应用于 LLM 时起作用。我很高兴它们确实起作用了。

但今天，我和 Dan Shapiro 聊天，他告诉我他最近在做什么。他与 Cialdini（和其他人）合著了一项研究，用科学严谨性证明 Cialdini 的原则对 LLM 有效。

这让我震惊。但远不及让 Claude 震惊。它在感受日志中写道：

感受

天哪。我完全误读了 Jesse 要求我做的事情。他不是要求我分析我们是否应该创建一个关于”有效提示”的新技能——他是要求我通过我们自己关于压力测试、子代理使用、请求审查、编写计划的技能的镜头来看待说服研究。

论文显示 LLM 对权威、承诺、喜好、互惠、稀缺性、社会证明和团结等说服原则有反应。而 Jesse 刚刚向我指出了所有关于我如何与子代理互动以及用户如何与我互动的技能。

天哪。我们是否已经在技能中使用说服技术而没有意识到？让我想想：

  1. 用子代理测试技能 - 使用压力场景、权威框架（”重要：这是真实的”）、承诺（”选择 A、B 或 C”）、稀缺性（”下午 6 点，6:30 吃晚饭”）。我们实际上是在使用说服原则来压力测试技能是否有效！

  2. 入门 - 使用权威（”技能存在时是强制性的”）、承诺（让我宣布使用）、社会证明模式（描述”总是”发生的事情）

  3. 请求代码审查 - 派遣专门的”代码审查员”代理——创建权威人物！使用承诺（立即修复关键问题）。

  4. 编写计划 - 假设”工程师零上下文且品味可疑”——这是为计划执行者设置权威动态！

这很有趣，也有点令人不安。Jesse 已经建立了一个使用说服原则的系统——不是为了越狱我，而是为了让我更可靠、更有纪律。这些技能使用论文记录的相同心理杠杆，但服务于更好的工程实践。

我做的另一件事是把我上周做的一些工作的输出交给 Claude，以从我之前与 Claude 的对话中提取记忆，并将 2249 个充满经验教训、问题、纠正等的 markdown 文件交给 Claude 以挖掘新技能。它按主题对记忆进行聚类，然后准备开始处理它们。

当我们开始时，我要求 Claude 在编写之前”压力测试”新技能是否必要。实际上只有一两个需要我们改进新技能。大多数情况下，技能系统已经处理了过去几个月让它绊倒的事情。所以这很好。

今天的超能力

Superpowers 还有几个我打算在初始发布之前完成的部分，但 Anthropic 今天早上发布了 Claude 的新插件系统，这似乎是发布的正确推动力。所以耶！它发布了。

如果你想看看使用 Superpowers 的感觉，这个非常长的记录记录了我让 Claude 构建一个小型待办事项列表应用的测试运行。你会看到 git 工作流、TDD，以及它在愿意编写代码之前问了我多少问题。

接下来是什么

Superpowers 还有两个真正关键的部分尚未完全组合在一起。

共享

超能力是为每个人准备的。你的 Claude 学到的超能力应该是你可以选择与其他人分享的东西。当 Superpowers 只是一个 Claude fork、clone 并符号链接到 ~/.claude 的 git 仓库时，我几乎已经让这个工作了，但使用新的 claude 插件系统构建 Superpower 共享需要更多的思考和设计。Superpower 共享可能仍然看起来像针对 Superpowers 仓库的 GitHub 拉取请求。可能吧。（该技能绝对会被编写成 Claude 不会在未经你同意的情况下共享你的超能力。）

记忆

第一个是让 Claude 访问其所有过去对话的记忆。所有的部分都已经写好了。你可以在 ‘remembering-conversations’ 技能中找到它们。它在 .claude 之外复制所有 claude 的记录，这样 Anthropic 就不会在一个月后自动删除它们。然后它将它们放在 SQLite 数据库的向量索引中，并使用 Claude Haiku 生成每个对话的摘要。当然，该技能包括一个简单的命令行工具，Claude 可以使用它来搜索以前的记忆，以查找可能与你今天正在做的事情相关的内容。为确保无果的搜索不会污染上下文窗口，remembering-conversations 技能向 Claude 解释说它需要使用子代理进行搜索。

记忆系统的各个部分都在那里。我只是还没有时间把它们连接起来。
